10 个实战 Skills

把 742 个传统色变成可执行的设计工作流

每个 skill 对应一个真实设计阻塞点：方向太虚、色板太散、落位困难、交付断层、读不清、品牌漂移、图表难分、旧稿难改、系列疲劳和印刷不确定。

Slash 触发 /xxd-*

输入对应命令，再补充项目场景、目标人群、已有色值、截图或生产限制。每个 skill 目录都自带完整 Markdown 清单和配色关系 CSV，推荐色可追溯到当前 742 色库。

01

xxd-color-brief

把模糊审美词变成可执行约束。

适合什么时候用

当需求只有“高级、东方、年轻、克制、不要俗”这类词时，先把它们转成冷暖、明度、饱和度、对比和风险边界。

怎么用

输入 `/xxd-color-brief`，补充载体、受众、参考图和想避开的气质。输出会给起始色、禁用方向、冲突提醒和下一步 skill。

02

xxd-palette-builder

从太多选择里筛出能用的色板。

适合什么时候用

当你有一个锚点色、HEX、情绪或场景，但不知道该配哪些颜色、谁做主色、谁只能点缀时。

怎么用

输入 `/xxd-palette-builder`，给出锚点和用途。输出会按可靠、识别、对比策略给主辅点缀、比例、取舍和风险。

03

xxd-palette-applier

把漂亮色板真正落到版面里。

适合什么时候用

当 swatches 看着不错，一放进海报、网页、PPT、包装或 UI 就开始互相抢时。

怎么用

输入 `/xxd-palette-applier`，提供色板和载体。输出会给阅读路径、区域角色、面积比例和误用清单。

04

xxd-ui-token

把传统色变成开发能接的 UI token。

适合什么时候用

当传统色要进入网站、App、后台、工具或设计系统，不能只交一串 HEX 给开发时。

怎么用

输入 `/xxd-ui-token`，说明产品类型和输出格式。默认生成 primitive、semantic、component、mode 四层 CSS variables。

05

xxd-accessible-color

让好看的传统色也能读得清。

适合什么时候用

当某个传统色很好看，但要拿来做正文、按钮、状态、焦点、图表或深色模式时。

怎么用

输入 `/xxd-accessible-color`，给出前景/背景或色板。输出会给 WCAG ratio、通过状态、同库修复色和非颜色线索。

06

xxd-brand-system

把一次配色沉淀成长期品牌规则。

适合什么时候用

当颜色要支撑官网、社媒、产品、包装、PPT 和长期识别，不能每次活动都重新发明时。

怎么用

输入 `/xxd-brand-system`，说明行业、价格带、竞品差异和渠道。输出会给锚点色、支撑色、比例、禁用和治理规则。

07

xxd-data-viz

按数据含义生成传统色图表方案。

适合什么时候用

当你要做分类图、连续色带、发散图、地图、仪表盘或数据故事，不能把海报配色直接搬进图表时。

怎么用

输入 `/xxd-data-viz`，给出图表类型、系列数量、背景和数据含义。输出会给分类、顺序、发散、高亮或语义色方案。

08

xxd-existing-design-audit

把旧稿和散乱 HEX 修成传统色系统。

适合什么时候用

当你不是从零开始，而是已有截图、CSS、Figma styles、品牌色或一堆近似 HEX 要整理。

怎么用

输入 `/xxd-existing-design-audit`，贴出现有色值或截图描述。输出会给盘点、根因诊断、保留/合并/替换/移除建议。

09

xxd-content-series

让系列内容既统一又不疲劳。

适合什么时候用

当你要做小红书、公众号、视频封面、课程或栏目，既要每期可识别，又不能每张都长一样时。

怎么用

输入 `/xxd-content-series`，说明平台、栏目、节奏和模板类型。输出会给固定层、变量层、模板层和轮换规则。

10

xxd-print-packaging

把屏幕色推进到包装和印刷生产。

适合什么时候用

当传统色要进入茶、香、书籍、礼盒、文创、标签、材质和印刷生产，不能只看屏幕 HEX 时。

怎么用

输入 `/xxd-print-packaging`，说明品类、材质、工艺、价格带和系列数量。输出会给面板用色、变体规则和打样检查清单。
